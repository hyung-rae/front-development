## 소프트웨어 공학

### 소프트웨어 공학

- 소프트웨어 위기를 극복 해보자
- 효율적 일정과 비용으로 품질 높은 소프트웨어 생산

### 소프트웨어 공학의 3R

- 재공학
  - 유지보수의 생산성 향상 소프트웨어 위기 극복
  - 부시고 다시짓는 x, 고쳐쓰자
  - 분석하고 재구성하고 역공한후 이관한다.
- 역공학
  - 소스코드를 통해서 역으로 문서 산출
- 재사용
  - 함수 와 객첵 , app 자체
  - 합성 중심
  - 생성 중심
- 개발단계
  - 계획, 분석, 설계, 구현, 테스트, 유지 보수
  - 가장 많은 비용이 발생하는 곳이 유지 보수

## 소프트웨어 개발 방법론

### 개발 방법론 종류

- 구조적
  - 하향식, 절차지향 언어
  - DFD, DD, Minispec, STD, ERD
- 정보공학
- 객체지향
  - class, 속성, method, 객체, 인스턴스, 메시지
  - 정보은닉, 캡슐화, 상속, 추상화, 다형성 (오버로딩, 오버라이딩)
- CBD (component based development)
- 애자일

### 개발 모델

- 폭포수 모델
- 프로토타이핑 모델
- 나선형 모델
  - 계획 → 위험분석 → 개발 → 평가
- RAD 모델
- V 모형
- 4세대 기법

### 애자일 방법론

- XP `의사소통 피드백 존중 용기 단순성`
- Scrum 스크럼 `스프린트, 백로그(기능 명세서)`

## 프로젝트 계획

### 프로젝트 핵심 관리 대상

- 3P 사람 문제 프로세스
- 개발 비용산정
  - 하향식 `사람이 하는것 전문가, 델파이`
  - 상향식 LOC `낙관치, 기대치(중간치), 비관치`
  - 수학적
    - COCOMO
      - 조직형 (5만 라인 이하)
      - 반분리형 (30만 라인 이하)
      - 내장형 (30만 라인 이상)
    - PUTNAM
    - FP
      - 내부 논리 파일
      - 외부 연계 파일 `입력, 조회, 출력`
- 개발 일정계획
  - WBS → PERT/CPM → 임계경로 → 간트차트

### 현행 시스템 분석

- `운영체제` `네트워크` `DBMS`

### CPND `contents platform network device`

### 미들웨어

- 원격 프로시저 호출 RPC

### 요구사항 개발 프로세스

- **도출 → 분석 → 명세 → 확인**
- 명세에는 **정형적, 비형적 명세** 기법 있음

### HIPO

- 세부적 도표

### 구조적 분석 도구

- DFD Data Flow Diagram
- DD Data Dictionary
- Minispec 소단위 명세서
- STD 상태전이도
- **ERD 객체 관계도**

### 객체지향 분석 방법론

- 럼바우 (객체 모델링, 동적 모델링, 기능 모델링)
- 부치 방법 (미시적 거시적 개발 프로세스)
- Jacobson 방법 (Use Case)
- Coad & Yourdon 방법 (ERD)
- Wirfs-Brock 방법 (분석과 설계에 명확한 구분이 없다)

## 소프트웨어설계의 기본원칙

### 소프트웨어 설계

- 요구사항 명세서 바탕으로 구체적인 설계서를 작성하는 단계

- 설계 종류

  - 상위설계
    - 아키텍처, 데이터, 인터페이스, 사용자 인터페이스
    - 분석 설계
  - 하위 설계
    - 모듈, 자료구조, 알고리즘
    - 실제 구현단에서 할 것들

- 설계의 원리
  - 분할과 정복
  - 추상화 (공통적인 것들을 묶어 놓은것)
    - **과정 추상화**
    - **데이터 추상화**
    - **제어 추상화**
  - 단계적 분해
  - 모듈화
  - 정보 은닉
  - 결합도와 응집도 (결합도는 낮추고 응집도는 높이는 것)

### 설계 모델링

- 변경이 용이하게
- 필요한 자료만 사용 효율성 높인다.
- 모듈 단위로 나누어 독립적으로 설계

- 구조 모델링
  - UML 정적 다이어그램 사용 **정적 구조**를 표현
- 행위 모델링

  - UML 동적 다이어그램 사용

- **협약**에 의한 설계
  - 선행 조건 : 오퍼레이션 사용 전에 반드시 참이 되어야 하는 조건
  - 결과 조건 : 오퍼레이션 사용 후 만족해야 하는 조건
  - 불변 조건 : 오퍼레이션이 실행되는 동안 항상 만족해야 하는 조건
